## 4. API‑Einrichtungs‑Guide

### Instagram Graph API / Facebook API
1. **Facebook Developer Account** erstellen: [developers.facebook.com](https://developers.facebook.com/)
2. **Neue App** anlegen (Typ: „Business“)
3. **Instagram Basic Display** bzw. **Instagram Graph API** hinzufügen
4. **App‑Berechtigungen** beantragen: `instagram_basic`, `pages_show_list`, `instagram_manage_insights`
5. **Test‑Benutzer** hinzufügen und in Live‑Modus versetzen
6. **Access Token** generieren – für Posts, Insights, Kommentare
7. **Offizielle Docs**: [Instagram Platform](https://developers.facebook.com/docs/instagram)

### Twitter / X API v2
1. **Twitter Developer Portal** auf [developer.twitter.com](https://developer.twitter.com/) öffnen
2. **Projekt** und **App** erstellen (Essential oder Elevated Access)
3. **API Keys & Tokens** generieren: `API Key`, `API Secret`, `Access Token`, `Access Token Secret`
4. **Berechtigungen** setzen (Read/Write/Like/DM)
5. **OAuth 2.0** für App‑only Authentication oder OAuth 1.0a für User Context
6. **Offizielle Docs**: [Twitter API Documentation](https://developer.twitter.com/en/docs)

### YouTube Data API v3
1. **Google Cloud Console** öffnen: [console.cloud.google.com](https://console.cloud.google.com/)
2. **Neues Projekt** erstellen oder auswählen
3. **YouTube Data API v3** aktivieren (Bibliothek → suchen → aktivieren)
4. **Credentials** erstellen: OAuth 2.0 Client ID (Web‑Application)
5. **Redirect‑URIs** konfigurieren (z.B. `http://localhost:8080`)
6. **Consent Screen** konfigurieren (externe Nutzer)
7. **Access Token** über OAuth‑Flow abrufen – für Uploads, Playlists, Analytics
8. **Offizielle Docs**: [YouTube Data API](https://developers.google.com/youtube/v3)
